## ToeplitzMatrix (2026-06-01)

New builtin `ToeplitzMatrix` (`src/linalg/toeplitzmat.c`), a matrix that is
constant along its diagonals. Attribute `Protected`. Symbol registered in
`sym_names.c`; docstring in `info.c`; registered in `linalg_init`
(`src/linalg/linalg.c`).

- `ToeplitzMatrix[n]` gives the `n x n` Toeplitz matrix whose first row and
  column are the successive integers `1..n`; the `(i, j)` entry is
  `|i - j| + 1`, so the matrix is symmetric.
- `ToeplitzMatrix[{c1, ..., cn}]` gives the `n x n` symmetric Toeplitz matrix
  whose first column (and first row) is the given list â€” plain symmetric, with
  no conjugation.
- `ToeplitzMatrix[{c1, ..., cm}, {r1, ..., rn}]` gives the `m x n` Toeplitz
  matrix with the first list down the first column and the second across the
  first row. The `(i, j)` entry is `c_{i-j+1}` when `i >= j`, and `r_{j-i+1}`
  otherwise.
- Entries are copied verbatim, so symbolic, exact, complex, machine and
  arbitrary-precision (e.g. ``1`20``) entries all flow through unchanged. For
  `m = n` the matrix is symmetric.
- The shared corner `c_1` should equal `r_1`; if not, the column element is
  used (the formula reads `c_1` on the diagonal, never `r_1`) and a
  `ToeplitzMatrix::crs` warning is emitted (the matrix is still built).
  `ToeplitzMatrix::argb` reports a zero-argument call; a first argument that is
  neither a positive integer nor a list, and over-arity calls, are returned
  unevaluated.
- Extensive unit tests in `tests/test_toeplitzmatrix.c` (integer / symbolic /
  complex square forms, wide and tall rectangular forms, the corner-mismatch
  warning path, symmetry and diagonal-constancy identities, `Dimensions`,
  machine / MPFR precision propagation, all unevaluated / diagnostic paths,
  attributes / docstring / interned symbol, and a repeated-evaluation leak
  stress loop). `leaks`-clean.

## VandermondeMatrix (2026-06-01)

New builtin `VandermondeMatrix` (`src/linalg/vandermondemat.c`), a matrix whose
rows are the successive powers of a sequence of nodes; the `(i, j)` entry is
`xi^(j-1)`. Attribute `Protected`. Symbol registered in `sym_names.c`;
docstring in `info.c`; registered in `linalg_init` (`src/linalg/linalg.c`).

- `VandermondeMatrix[{x1, ..., xn}]` gives the `n x n` Vandermonde matrix on the
  nodes `xi`; `VandermondeMatrix[{x1, ..., xn}, k]` gives the `n x k` matrix.
- The first column is all ones, the second is the nodes, the third their
  squares, and so on. Entries are built as `Power[xi, j-1]` and simplified by
  the evaluator, so symbolic nodes stay as `Power` expressions while numeric
  powers fold to their value. The first column is the literal integer `1`, so a
  zero node reads as `1` there (not `Indeterminate`), matching the
  interpolation semantics.
- The nodes need not be numerical or distinct. Precision flows from the nodes
  themselves (e.g. ``2`20``) or from wrapping the result in `N`. For distinct
  nodes, `LinearSolve[V, b]` recovers the coefficients of the interpolating
  polynomial through `{xi, bi}`, and `Det[V]` is the product of node
  differences `prod_{i<j} (xj - xi)`.
- `VandermondeMatrix::argt` reports a zero-argument call. An empty node list, a
  non-list first argument, a non-positive integer `k`, the single-argument
  structured-array conversion form `VandermondeMatrix[vmat]` (unsupported â€”
  Mathilda has no structured-array representation), and over-arity calls are
  returned unevaluated.
- Extensive unit tests in `tests/test_vandermondematrix.c` (symbolic / numeric
  square forms, single node, rectangular `n x k` forms, the `0^0 = 1` /
  all-ones-first-column convention, `Dimensions`, numeric and symbolic
  determinant identities, `LinearSolve` interpolation, machine / MPFR precision
  propagation, all unevaluated / diagnostic paths, attributes / docstring /
  interned symbol, and a repeated-evaluation leak stress loop). `leaks`-clean.

## HankelMatrix (2026-06-01)

New builtin `HankelMatrix` (`src/linalg/hankelmat.c`), a matrix that is
constant along its antidiagonals. Attribute `Protected`. Symbol registered in
`sym_names.c`; docstring in `info.c`; registered in `linalg_init`
(`src/linalg/linalg.c`).

- `HankelMatrix[n]` gives the `n x n` Hankel matrix whose first row and column
  are the successive integers `1..n`, with zeros below the main antidiagonal.
- `HankelMatrix[{c1, ..., cm}]` gives the `m x m` Hankel matrix whose first
  column is the given list (zeros below the antidiagonal).
- `HankelMatrix[{c1, ..., cm}, {r1, ..., rn}]` gives the `m x n` Hankel matrix
  with the first list down the first column and the second across the last
  row. The `(i, j)` entry is `c_{i+j-1}` when `i + j - 1 <= m`, and
  `r_{i+j-m}` otherwise.
- Entries are copied verbatim, so symbolic, exact, complex, machine and
  arbitrary-precision (e.g. ``1`20``) entries all flow through unchanged; the
  antidiagonal fill is the exact integer `0`. For `m = n` the matrix is
  symmetric.
- The shared corner `c_m` should equal `r_1`; if not, the column element is
  used and a `HankelMatrix::crs` warning is emitted (the matrix is still
  built). `HankelMatrix::argb` reports a zero-argument call; a first argument
  that is neither a positive integer nor a list, and over-arity calls, are
  returned unevaluated.
- Extensive unit tests in `tests/test_hankelmatrix.c` (integer / symbolic /
  complex square forms, wide and tall rectangular forms, the corner-mismatch
  warning path, symmetry and antidiagonal-constancy identities, `Dimensions`,
  machine / MPFR precision propagation, exact-zero fill, all unevaluated /
  diagnostic paths, attributes / docstring / interned symbol, and a
  repeated-evaluation leak stress loop). `leaks`-clean.

## HilbertMatrix (2026-06-01)

New builtin `HilbertMatrix` (`src/linalg/hilbertmat.c`), the `m x n` Hilbert
matrix whose `(i, j)` entry is `1/(i + j - 1)`. Attribute `Protected`.
Symbol registered in `sym_names.c`; docstring in `info.c`; registered in
`linalg_init` (`src/linalg/linalg.c`).

- `HilbertMatrix[n]` gives the `n x n` Hilbert matrix; `HilbertMatrix[{m, n}]`
  gives the `m x n` matrix. Entries are exact `Rational`s by default.
- The `WorkingPrecision` option selects the entry representation:
  `Infinity` (default) â†’ exact `Rational`s; `MachinePrecision` â†’ machine
  `Real`s; a digit count `d` â†’ `d`-digit MPFR `Real`s (degrading to machine
  precision when `d` is at or below machine precision, or when built without
  MPFR).
- Diagnostics leave the call unevaluated: `::argx` (0 arguments), `::dims`
  (dimension spec that is not a positive integer or a pair of positive
  integers), and `::nonopt` (trailing non-option arguments).
- Extensive unit tests in `tests/test_hilbertmatrix.c` (exact square /
  rectangular matrices, symmetry, `Transpose` dimension swap, known
  determinants and the exact integer inverse, machine / MPFR precision
  paths, all diagnostic paths, attributes / docstring / interned symbol,
  and a repeated-evaluation leak stress loop). Valgrind/`leaks`-clean.

## Ratios (2026-06-01)

New builtin `Ratios` (`src/list.c`), the multiplicative analog of
`Differences` and inverse of `FoldList[Times, ...]`. Attribute `Protected`.
Symbol registered in `sym_names.c`; docstring in `info.c`.

- `Ratios[list]` gives the successive ratios `{l[[2]]/l[[1]], l[[3]]/l[[2]],
  ...}`, of length `s - 1`. Ratios divides successive elements by preceding
  ones, so `Ratios[{x1, x2}]` gives `{x2/x1}`.
- `Ratios[list, n]` applies the ratio operator `n` times, giving length
  `s - n`. `n` must be a non-negative integer (`n = 0` returns `list`
  unchanged); e.g. `Ratios[{a, b, c, d, e}, 2]` is
  `{(a c)/b^2, (b d)/c^2, (c e)/d^2}`.
- `Ratios[list, {n1, n2, ...}]` gives the successive `nk`-th ratios at level
  `k` of a nested list, and is equivalent to
  `Ratios[Ratios[list, n1], {0, n2, ...}]`. Each `nk` must be a non-negative
  integer.
- Division threads element-wise over sublists via the `Listable` `Power`/`Times`
  that `Divide` rewrites to, so for a matrix `m`, `Ratios[m]`
  (= `Ratios[m, 1]` = `Ratios[m, {1, 0}]`) takes ratios of successive rows
  within each column, while `Ratios[m, {0, 1}]` takes ratios of columns within
  each row.
- The head of the input is preserved; a list of length `<= n` reductions yields
  the empty list. Works on machine integers, GMP arbitrary-precision integers
  (exact `Rational` results), machine-precision doubles, and symbolic
  expressions. First ratios are constant for a geometric sequence, and
  `FoldList[Times, x, Ratios[list]]` reconstructs the original list.

Implemented by mirroring the `Differences` helpers: `ratio_divide`,
`ratio_once`, `ratio_n`, and `ratio_levels` build `Divide[hi, lo]` and let the
evaluator reduce it. 28 unit tests in `tests/test_ratios.c`; valgrind/`leaks`
clean.
