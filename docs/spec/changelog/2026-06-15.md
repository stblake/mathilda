## Feature: multi-generator radical-rational normal form in `Simplify` (2026-06-15)

`Simplify` could not combine a rational function of two or more distinct radical
generators when collapsing it required the relations between them. The reported
case was the round-trip check on the radical integrator:

```
In[1]:= r = Integrate[1/(x^3 (a + b x)^(1/3)), x];
In[2]:= D[r, x] // Simplify
```

`D[r, x]` is a four-term sum of fractions over `a^(k/3)` and `(a+b x)^(k/3)`;
`Together`/`Cancel` handle a single radical generator but left these separate
because the denominators are different fractional powers of *two* bases. The
collapse needs reduction modulo the generator relations `s^3 = a`,
`t^3 = a+b x`.

New pass `simp_radical_rational` (`src/simp/radrat.c`, wired into `simp_dispatch`
in `src/simp/simp_search.c`, mirroring `simp_trig_rational`) works in the
quotient ring `K[g_1, вЂ¦, g_n] / <g_k^{q_k} в€’ base_k>`:

- **Detect** the distinct radical bases `base_k` (each `base_k^(p/q_k)`,
  `q_k > 1`); require at least two and every base **provably positive** (so
  `base_k^(p/q) в†’ g_k^p` is branch-safe). Single-generator and
  negative/complex-numeric-base inputs are left to the existing path.
- **Substitute** outer-base-first to fresh polynomial generators
  (`poly_subst_radical_to_gen`), build the relation `g_k^{q_k} в€’ V_k` for each
  compound base (where `V_k` expresses `base_k` in the other generators), keeping
  only relations that stay inside the substituted ring.
- **Together**, reduce numerator and denominator modulo the relation ideal
  (`PolynomialRemainder`), rationalise the denominator (`PolynomialExtendedGCD`),
  substitute the radicals back, and finish with `Cancel[N / Factor[D]]`.
- **Strict score gate** вЂ” the result is returned only when its `SimplifyCount`
  beats the input, so the pass can never regress a case; it returns NULL
  otherwise and `simp_dispatch` continues unchanged.

Result: `D[Integrate[1/(x^3 (a+b x)^(1/3)), x], x] // Simplify` now returns
`1/(x^3 (a+b x)^(1/3))`. Scope is `Simplify` only; `Together`/`Cancel` keep their
single-generator behaviour. `src/simp/radrat.c` is added to the makefile glob and
to `tests/CMakeLists.txt` `COMMON_SRC`.

## Fix: post-integration result normalisation (`Integrate`) (2026-06-15)

`Integrate[1/(x^3 (a + b x)^(1/3)), x]` (and the linear/quadratic/ratio
radical family generally) returned a correct but needlessly verbose
antiderivative вЂ” unsimplified rational pieces over denominators in the
compound radicand and stray `(1/a)^(k/3)` constants. A post-method
normalisation pass, `intsimp_finalize` (`src/calculus/intsimp.c`, wired at the
one chokepoint in `builtin_integrate`, `src/calculus/integrate.c`), now cleans
**radical-bearing** antiderivatives (those carrying a `Power[f(x), p/q]` of the
integration variable). Non-radical / partially-closed (`nested Integrate`)
results pass straight through untouched вЂ” the deficiency only manifests in the
substitution integrators, and reshaping rational/Risch output perturbed forms
the corpus verifier pins. The pass:

- **x-free reciprocal/product powers.** `(1/a)^(2/3) в†’ a^(-2/3)` etc. via
  `PowerExpand` applied only to x-free `Power[c, p/q]` subexpressions whose base
  is positive under the symbol-positive convention (`it_normalize_xfree_powers`
  + `it_pos_base`) вЂ” the reason `a^(7/3)` denominators and clean `Log`/`ArcTan`
  arguments now appear. The positive-base guard is essential: PowerExpand on a
  negative/complex numeric constant (`(-3)^(1/2)`) would cross a branch cut and
  silently change a coefficient's value.
- **Algebraic recombination.** The back-substituted Hermite sum is combined and,
  crucially, its `Numerator` / `Denominator` are `Expand`-ed so integer-power
  compounds collapse in x (`6 a^4 в€’ 12 a^3 (a+b x) + 6 a^2 (a+b x)^2 в†’
  6 a^2 b^2 x^2`) before a final `Cancel` (`it_recombine_algebraic`). Gated to
  replace the original only when it strictly shrinks (leaf count), size-capped
  to avoid multivariate-GCD blow-up.
- **Inverse-trig arguments** are distributed and sign-normalised
  (`ArcTan[-(p+q)] в†’ -ArcTan[p+q]`), distributing the negation so ArcTan's own
  oddness does not re-pull it.

The reported case now returns
`(4 (a+b x)^(5/3) в€’ 7 a (a+b x)^(2/3))/(6 a^2 x^2) + (2 b^2 ArcTan[1/Sqrt[3] +
2 (a+b x)^(1/3)/(Sqrt[3] a^(1/3))])/(3 Sqrt[3] a^(7/3)) в€’ вЂ¦`, matching
Mathematica up to an immaterial `Log` sign (constant of integration).
Correctness re-verified with `PossibleZeroQ[D[result, x] в€’ integrand]`.
`intsimp_finalize` takes ownership of the result and is a single tree-walk for
the non-radical fast path; no new builtins or attributes; prototype added to
`intsimp.h`. Regression-checked: all integration unit suites pass, the
`IntegrateRationalTests.m` corpus is unchanged (в‰¤ baseline diffs, no new
crashes), and valgrind shows no new leaks vs the `Sin[1.0]` baseline.

## Feature: nested-radical composition over symbolic bases in `Power` (2026-06-15)

`Sqrt[(1/27/a)^(2/3)]` now reduces to `1/3 (1/a)^(1/3)` (previously left
unevaluated). Two principal-branch-sound rules were added to `builtin_power`
in `src/power.c`:

- **Power-of-Power composition for any base when `|inner exponent| < 1`.** The
  guard on `(B^r)^s в†’ B^(rВ·s)` now also fires when `r = p/q` is a non-integer
  rational with `|p| < q`, for *any* `B` (no positivity needed): `|r| < 1` keeps
  `rВ·Arg(B)` inside `(-ПЂ, ПЂ]`, so the intermediate power crosses no branch cut.
  This collapses e.g. `Sqrt[a^(2/3)] в†’ a^(1/3)` and `Sqrt[Sqrt[a]] в†’ a^(1/4)`.
  New helper `inner_exp_abs_lt_one`. Inner exponents with `|p| в‰Ґ q` (`a^(3/2)`)
  and symbolic inner exponents (`2^a`) still stay unevaluated.
- **Positive-coefficient split for mixed `Times` bases.** `(cВ·w)^(p/q) в†’
  c^(p/q)В·w^(p/q)` when `c > 0` is a numeric factor that fully reduces under `q`
  and `w` is the symbolic residual (sound for any `w` because `Arg(c) = 0`). New
  helper `is_positive_numeric`; gated on `factor_fully_reduces_under_q` so
  cosmetic splits do not happen вЂ” `Sqrt[2 Pi]`, `(4 Pi)^(2/3)`,
  `Sqrt[2 Sqrt[3]]` remain nested.

Behavioral note: `Nest[Sqrt, a, 3]` now folds to `a^(1/8)` (was nested
`Power`s); its test was updated. Coverage added in
`tests/test_power_corpus.c` (`test_power_nested_symbolic_compose`). No new
memory leaks (valgrind identical to baseline).

## Feature: `AiryAi` вЂ” Airy function Ai(z) (2026-06-15)

Implemented `AiryAi` in `src/special_functions/airyai.c`, with its derivative
head `AiryAiPrime`. `AiryAi[z]` is the Airy function Ai(z), the solution of
yвЂі = z y that decays as z в†’ +в€ћ вЂ” an entire function of z.

- **Exact values.** `AiryAi[0] = 1/(3^(2/3) Gamma[2/3])` and
  `AiryAi[В±Infinity] = 0`. Other exact arguments (e.g. `AiryAi[2]`) stay
  unevaluated and numericalize only under `N`.
- **Unified numeric core.** A file-local complex-MPFR toolkit (`acx`, pairs of
  `mpfr_t`, the `ecx`/`pcx` pattern) computes Ai(z) and AiвЂІ(z) together for real
  and complex inputs at machine and arbitrary (MPFR) precision. It routes on
  `r = |z|` and target precision `P`:
  - **Maclaurin series** for small/moderate `r`: recurrence
    `a_n = a_{n-3}/(n(n-1))` from AiвЂі = z Ai, with `(2/3) r^{3/2}/ln2` guard
    bits absorbing the partial-sum cancellation. The closed-form constants
    `a_0 = Ai(0)`, `a_1 = AiвЂІ(0)` are formed with the `2/3`, `1/3` fractions
    built exactly in MPFR (not via doubles).
  - **Asymptotic series** (DLMF 9.7.5) for large `r`: О¶ = (2/3) z^{3/2},
    `u_k = (6k-5)(6k-3)(6k-1)/((2k-1)В·216В·k) u_{k-1}`, summed to optimal
    truncation. Near the negative real axis (2ПЂ/3 < |arg z| в‰¤ ПЂ) the DLMF 9.2.12
    connection relation `Ai(z) = -[П‰ Ai(П‰z) + П‰М„ Ai(П‰М„z)]`, П‰ = e^{2ПЂi/3}, maps
    the argument into two well-conditioned in-sector points so the oscillation
    emerges with no catastrophic cancellation.

    against MPFR's `mpfr_ai` over a half-integer grid in the unit tests.
  - Machine-precision results whose magnitude overflows the double range (e.g.
    `AiryAi[150. I] в‰€ 8.1951Г—10^374 + вЂ¦`) are promoted to MPFR rather than
    becoming `Inf`.

  Examples: `AiryAi[1.8] = 0.0470362`, `AiryAi[2.5 + I] = -0.00191209 -
  0.0180329 I`,
  `N[AiryAi[2], 50] = 0.034924130423274379135322080791807609761060213897583`.
- **Derivatives** (`src/calculus/deriv.c`). `D[AiryAi[z], z] = AiryAiPrime[z]`
  and `D[AiryAiPrime[z], z] = z AiryAi[z]`, so higher derivatives reduce

- **Taylor series at 0** comes from the generic Taylor-via-D path once the exact
  values at 0 exist: `Series[AiryAi[x], {x, 0, 7}]` reproduces the closed-form
  coefficients.
- **Asymptotic series at в€ћ** (`src/calculus/series.c`,
  `try_series_airyai_at_infinity`). `Series[AiryAi[x], {x, Infinity, n}]` returns
  `Times[Exp[-(2/3) x^(3/2)], SeriesData[1/x, вЂ¦]]` вЂ” a Puiseux series in
  `(1/x)^(1/4)` with the essential-singularity prefactor kept symbolic, matching
  Mathematica (e.g. `E^(-2/3 x^(3/2)) ((1/x)^(1/4)/(2 Sqrt[Pi]) -
  5 (1/x)^(7/4)/(96 Sqrt[Pi]) + O[1/x]^(13/4))`).
- **`AiryAiPrime`** is a minimal head: it has the derivative rule and the exact
  value `AiryAiPrime[0] = -1/(3^(1/3) Gamma[1/3])` (needed by the series) but no
  general numeric evaluator (`N[AiryAiPrime[2.0]]` stays unevaluated).
- **Attributes.** `AiryAi` and `AiryAiPrime`: `Listable`, `NumericFunction`,
  `Protected`, `ReadProtected`. Listable threading gives
  `AiryAi[{1.2, 1.5, 1.8}] = {0.106126, 0.0717495, 0.0470362}`.
- **Symbols / docs.** `SYM_AiryAi`, `SYM_AiryAiPrime` registered in
  `sym_names.c`; docstrings in `info.c`. Extensive unit + oracle + stress
  coverage in `tests/test_airyai.c`; no memory leaks (verified with macOS
  `leaks`).
