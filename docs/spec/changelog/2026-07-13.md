# Changelog: week of 2026-07-13 (Mon) – 2026-07-19 (Sun)

## Graphics: Animate[expr, {t, tmin, tmax}, opts...] (2026-07-13)

Added the `Animate` function, which opens an interactive Raylib window and
animates an expression by re-evaluating it at each frame with the iterator
variable bound to the current parameter value.

### What was added

**New files:**
- `src/graphics/animate.h` — public header declaring `builtin_animate`
- `src/graphics/animate.c` — full implementation of the animation window and
  playback loop

**Modified files:**
- `src/graphics/render.c` — added `graphics_render_in_region(expr, rx, ry, rw, rh)`,
  a reusable draw function that renders a `Graphics[...]` object into a
  sub-region of an already-open Raylib window (used by `Animate`'s per-frame draw)
- `src/graphics/render.h` — declared `graphics_render_in_region`
- `src/graphics/graphics.h` — included `animate.h`
- `src/graphics/graphics_init.c` — registered `Animate` (HoldAll, Protected) and
  all option/value keywords as inert symbols with docstrings
- `src/sym_names.h` / `src/sym_names.c` — added `SYM_Animate`,
  `SYM_AnimationDirection`, `SYM_AnimationRate`, `SYM_AnimationRepetitions`,
  `SYM_AnimationRunning`, `SYM_AppearanceElements`, `SYM_DefaultDuration`,
  `SYM_ControlPlacement`, `SYM_RefreshRate`, `SYM_Forward`, `SYM_Backward`,
  `SYM_ForwardBackward`, `SYM_BackwardForward`

### Options implemented

| Option | Behavior |
|--------|---------|
| `AnimationDirection` | `Forward` (default), `Backward`, `ForwardBackward`, `BackwardForward` |
| `AnimationRate` | parameter units per second; overrides `DefaultDuration` |
| `AnimationRepetitions` | integer or `Infinity` (default) |
| `AnimationRunning` | `True` (default, starts playing) or `False` (starts paused) |
| `AppearanceElements` | `All`, `None`, or a list of named element strings |
| `DefaultDuration` | seconds for one full pass (default 1.0) |
| `ControlPlacement` | `Bottom` (default) or `Top` |
| `RefreshRate` | target display FPS (default 60) |

### Control bar

The control bar has two regions (order reflects `ControlPlacement`):

**Labeled variable slider rows** — one row per iterator `{var, min, max}`:
- Variable name label on the left (e.g. `a`)
- Range hints (`tmin` / `tmax`) flanking the track
- Filled track + circle handle showing the current position
- Current numeric value on the right
- Draggable: click or drag anywhere on the track to scrub; pauses animation
  and phase-syncs all other iterators from the dragged one

**Playback button row** (below slider rows):
- **R** — reset to start
- **⏮ / ⏭** — step backward/forward by 5% of the range
- **▶ / ⏸** — play / pause
- **(direction icon)** — cycle through `Forward` → `Backward` →
  `ForwardBackward` → `BackwardForward` → …
- **− / +** — halve / double playback speed; multiplier shown beside them

Keyboard shortcuts: `Space` (play/pause), `←`/`→` (step 2% of range),
`R` (reset), `Esc` (close).

Multiple iterators (e.g. `Animate[f[a, b], {a, 0, 5}, {b, -1, 1}]`) each
get their own labeled slider row; all are phase-synced so they advance
together during playback.

## Graphics: ComplexPlot and ComplexPlot3D (2026-07-14)

Added `ComplexPlot` and `ComplexPlot3D` for visualising complex-valued
functions `f(z)` over a rectangular domain in the complex plane.

### ComplexPlot

Domain-colouring 2D plot.  Each grid cell is coloured by `Arg(f(z))` via
the **thermal ramp** (matching `DensityPlot`/`Plot3D` defaults), with
brightness `|f(z)|/(1+|f(z)|)` so the origin fades to black.  Returns
`Graphics[...]`.

**Signature:** `ComplexPlot[f, {z, zmin, zmax}, opts...]`

`zmin`/`zmax` may be complex numbers (`-2-2*I`), pure reals, or symbolic
complex bounds; their Re/Im parts define the rectangular x–y domain.  Both
forms `Complex[re, im]` and `Plus[re, Times[im, I]]` are accepted after
evaluation.

### ComplexPlot3D

3D surface where **height = `|f(z)|`** and **colour = `Arg(f(z))`** via the
thermal ramp.  Returns `Graphics3D[...]` rendered in the orbit-camera window.
`Lighting -> None` is recommended for accurate phase colours (disables
Lambertian shading).

### Shared options (both functions)

| Option | Default | Notes |
|--------|---------|-------|
| `PlotPoints` | `50` | grid resolution per axis |
| `ColorFunction` | thermal by arg | `f[re, im] → color`, or named ramp (`"Rainbow"`, `"CoolTones"`, etc.) keyed to normalised `Arg` |
| `ColorFunctionScaling` | `True` | normalises `re`/`im` to `[0,1]` before custom `ColorFunction` |
| `RegionFunction` | `None` | `f[x,y]` mask |
| Standard Graphics/3D options | — | pass through |
